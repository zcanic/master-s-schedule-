# Void KV API 说明文档

这是一个基于 Cloudflare Workers 和 KV (Key-Value) 存储构建的极简持久化后端。它被设计为一个“虚空广播站”，通过简单的暗号（Key）来存取数据。

## 1. 基础信息

- **API 根地址**: `https://kvapi.zc13501500964.workers.dev`
- **存储技术**: Cloudflare KV (高性能分布式键值存储)
- **跨域策略**: 全开放 (`CORS: *`)，支持任何 Web 前端调用。

## 2. 接口协议

本 API 采用路径映射逻辑：`URL 路径` 即为 `存储暗号 (Key)`。

### 📡 向虚空广播 (写入数据)
- **方法**: `POST`
- **路径**: `/{你的暗号}`
- **要求**: 
    - 暗号长度必须 ≥ 3 位。
    - 请求体 (Body) 为你想存入的字符串（支持 JSON 字符串）。
- **返回**: `Signal Received. Void Updated. (数据已写入)`

### 📥 从虚空读取 (获取数据)
- **方法**: `GET`
- **路径**: `/{你的暗号}`
- **返回**: 
    - 如果暗号存在，返回存储的原始字符串。
    - 如果暗号不存在，返回空响应 (null)。

## 3. 错误处理

| 状态码 | 含义 | 常见原因 |
| :--- | :--- | :--- |
| **400** | 暗号无效 | 路径为空或暗号长度不足 3 位。 |
| **405** | 方法禁用 | 使用了除 GET, POST, OPTIONS 以外的方法。 |
| **500** | 虚空崩塌 | KV 绑定异常或 Cloudflare 内部错误。 |

## 5. 快速调用示例 (cURL)

```bash
# 存入数据
curl -X POST https://kvapi.zc13501500964.workers.dev/my_key -d "Hello Void"

# 读取数据
curl https://kvapi.zc13501500964.workers.dev/my_key
```
